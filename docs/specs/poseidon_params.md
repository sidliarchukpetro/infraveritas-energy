## 1. Decision

**Використовуємо Poseidon (Hades design) на BN254 з Circom-compatible параметрами через Noir's `std::hash::poseidon::bn254` module.**

Це **де facto standard** Poseidon для BN254 — ті самі параметри що:
- Noir stdlib `std::hash::poseidon::bn254::hash_N`
- Circom's `circomlib/poseidon.circom`
- Aragon ZK research reference implementation
- IOTA / Sui blockchain Poseidon precompiles
- Aztec, Polygon zkEVM, Mina (mainnet)

**Не Poseidon2** — хоча Poseidon2 дає ~5% менше constraints, ми платимо це за широкий cross-language library support. Critical для трьох-компонентної consistency requirement.

## 2. Parameters (frozen)

| Parameter | Value |
|---|---|
| Hash function | Poseidon (Hades design) |
| Curve | BN254 (alt_bn128) |
| Field size | 21888242871839275222246405745257275088548364400416034343698204186575808495617 |
| S-box exponent (α) | **5** (degree-5 power map) |
| Constants source | Circom-compatible (matches Noir stdlib) |
| State sizes (t) | **2 through 17** (Noir provides hash_1 through hash_16 inputs) |
| Full rounds (r_F) | **8** (4 initial + 4 final) |
| Partial rounds (r_P) | **calibrated per t** per Hades paper §3.2 |
| MDS matrix | Circom-compatible (different per t) |
| Round constants | Circom-compatible (different per t, total = (r_F + r_P) × t values) |

**Critical note on r_P:** Partial round count is **calibrated per state size**, not fixed. Use whatever Noir's `std::hash::poseidon::bn254` provides natively for the chosen t. Common values: t=2 → 56, t=3 → 57, t=4 → 56.

## 3. Hashing construction for V3 canonical payload

V3 canonical payload = 56 metadata bytes + 100 readings × 24 bytes = **2456 bytes**.

### Field element encoding

Pack each uint64 into a separate BN254 Field element (Field holds 254 bits, accommodates uint64 trivially).

- Metadata: 7 fields (device_id, session_id, epoch_start_ts, lat_e7, lon_e7, light_level, tamper_flag)
- Readings: 300 fields (100 × 3 values per reading)
- **Total: 307 field elements**

### Construction options

**Option A (sponge with t=3, rate=2, capacity=1):**
- Absorb 2 field elements per permutation
- ~154 permutations
- Approximate constraint cost: ~154 × 2098 = ~323,000 constraints just for hashing
- Simple, well-understood, matches Aragon Noir reference

**Option B (Merkle tree of hash_2):**
- Build binary tree over 307 leaves
- log₂(307) ≈ 9 levels, ~512 hash_2 calls
- Approximate constraint cost: ~512 × 2098 = ~1,074,000 constraints — worse

**Option C (batching via hash_16):**
- 307 / 16 = 20 hash_16 calls + final combiner
- Approximate constraint cost: complex, depends on hash_16 internals — TBD

**Decision: Option A (sponge t=3)** — simplest, lowest constraints, easiest to reproduce cross-language.

**Implementation:** Sponge construction defined per Hades paper §3. Reference: Aragon ZK research blog (linked in §7) provides exact algorithm for sponge with t=3 absorbing rate-2.

## 5. Cross-language implementation

Three components must produce identical hash output. Implementation per component:

### Noir circuit (v08)

```rust
use dep::std::hash::poseidon;
let h = poseidon::bn254::hash_2([a, b]);
// For sponge:
// Manual sponge construction або use third-party library like noir-poseidon від TaceoLabs
```

**Dependency:** Noir compiler ≥ 0.34.0 (per noir-lang/poseidon).

### Edge Python (`edge/hal/canonical.py`)

Replace placeholder. Library candidates (verify against test vector #5 before committing):

1. **`poseidon-hash` PyPI** — pure Python, verify Circom compatibility
2. **`circomlibjs` via subprocess** — Node.js bridge, ugly but guaranteed match
3. **Custom port from circomlib JS source** — ~200 lines, full control

**Selection criteria:** must pass all 5 test vectors from §4.

### Aggregator TypeScript

**Library:** [`circomlibjs`](https://github.com/iden3/circomlibjs) `poseidon` — native fit, identical to Circom params we specified.

```typescript
import { buildPoseidon } from "circomlibjs";
const poseidon = await buildPoseidon();
const h = poseidon([a, b]);
const hashBytes = poseidon.F.toString(h);
```

### Phase 1: Generate authoritative test vectors (1-2 hours)

3. Run `nargo execute`, capture outputs

### Phase 2: Edge migration (30 min, depends Phase 1)

1. Add Python Poseidon lib до `edge/pyproject.toml`
2. Replace SHA-256 у `edge/hal/canonical.py::compute_payload_hash`
3. Update test → `test_matches_poseidon_reference` (uses test vectors)
4. Verify all 5 vectors pass
5. Commit

### Phase 3: v08 circuit (3-5 days, depends Phase 2)

1. Implement canonical encoding у Noir matching Python `canonicalize()`
2. Apply Poseidon hash using frozen params
3. Output `payloadHash` як public input
4. Compile, generate HonkVerifier.sol
5. Commit з reference proof generation script

### Phase 4: Aggregator integration (depends Phase 3)

1. Implement Poseidon hash через circomlibjs
2. Verify match з edge-computed hash
3. Wire прокинення hash до circuit як public input
4. End-to-end test через V3 submitProof
